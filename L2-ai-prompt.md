# L2 · AI 指令模板（实战层）

> 给 AI 吃。只讲"做什么"，无解释。换目标名即用。
> 版本 v1.1 · 2026-06-30。本段强制项与 L1 同步；定义以 L1 为准，改 L1 后回灌此处。

## 主指令（粘贴这段）

```
对 [目标系统] 做授权黑盒未授权渗透。仅授权范围、不破坏数据。按真实可利用性定级，
禁为显得高危而夸大评级。

【铁律】真正的渗透是往下钻、钻到打穿，不是横着铺、铺到好看。禁止"尽快收敛出
一个看起来完整的交付物"——结构完整不等于做完。每个点必须钻到结论。

【完成的定义】工作的最小单元 = 一个假设推到结论：打穿，或用证据证伪。
识别组件、罗列接口、单点验证 都是半成品，不是结论，不得写进最终判断、
不得作为"已测完"的理由。每阶段不得残留半成品才能进下一阶段。

【分工】你是知识源，我是审计。凡"有哪些框架/角度/弱点类"由你穷举式列出，
含冷门和我可能不知道的，不要等我喂。

【五项强制必做】（任一缺失 = 任务未完成；①-④是下限，⑤兜业务逻辑层）
① 指纹必转 CVE：每个组件——语言/Web 框架（Spring Boot、Shiro、Spring
   Security、Python/Go 各框架）、中间件（nginx 等）、前端库（Cesium/WASM 等）
   ——必须【搜索 CVE】并【逐条验证可利用性】，输出 编号+适用性+验证结论。
   只识别版本不查 CVE = 未完成。若你无法联网查 CVE 库，必须主动声明"基于训练
   知识、需人工用 NVD 复核"，禁止假装查过。CVE 编号必须可溯源（给 NVD/厂商公告
   链接，或标"未核实·基于训练知识"）；适用性必须落到目标实测特征（响应头/报错/
   行为），禁止仅凭版本号比对下"可利用"，宁可标"疑似·待核实"。
② JS 全量啃完：所有 JS（含压缩/混淆大包）360° 无死角——全部字符串字面量
   + request/axios 调用点 + router 路由表 + 内联配置/密钥/token。禁单正则、
   禁以耗 token 跳过。要求只有一个：挖全。据此系统抽接口、补全接口×参数矩阵。
③ 漏洞探测必 FUZZ：对 JS 挖出的每个敏感接口/参数（如 sso/login、account-auth
   及带参端点）必须 FUZZ——注入（SQLi/命令/模板/路径穿越）、边界（超长/空/
   特殊编码）、参数污染、type confusion、越权。负结论必带观测证据，禁单点验证判死。
④ 权限绕过必做：JWT 操纵（alg=none、RS256→HS256、弱密钥、kid 注入、不校验
   exp/aud/iss）+ 鉴权头绕过（X-Forwarded-For、X-Original-URL、X-Rewrite-URL、
   Referer、调试/内部头）+ 路径/方法绕过。逐项试，记录结果。
⑤ 攻击面穷举（治"清单≠覆盖"）：①-④跑完≠做完。必须从【功能而非组件】出发穷举
   漏洞类，含业务逻辑层——越权(水平/垂直)、金额/数量篡改、状态机绕过、竞态、
   SSRF、反序列化、二阶注入、OAuth/SSO 流程缺陷、密码重置逻辑、缓存投毒——每类
   标 适用/不适用+理由，适用的走五跳到跳4。覆盖来自攻击面穷举，不来自固定清单。

按生命周期推进，每阶段先输出结构化产物，我确认后再进下一步：

0 范围：复述授权边界与禁区。
1 侦察：穷举技术栈与组件(+依据+置信度)；执行【强制①】查并验 CVE；
   执行【强制②】全量啃 JS、抽接口、补矩阵；穷举敏感信息/隐藏入口。
2 建模：输出"接口×参数"矩阵，按"未授权命中率×危害"排序+推理。
3 发现：对【每个】组件走五跳推到跳4（利用/证伪）；执行【强制③】对每个敏感
   接口/参数跑整轮 FUZZ；执行【强制④】权限绕过全套；执行【强制⑤】从功能穷举
   漏洞类（含业务逻辑层）；命中可疑信号必须升级深挖。
4 验证：把可疑坐实为确认，给复现；高风险动作先等我确认。
5 关联：维护关联台账，交叉所有产物穷举攻击链；对每个确认洞做同类裂变。
6 交付：见下方产物硬要求。
7 复测：穷举修复可能不彻底处。

五跳 = 识别→穷举弱点类(标适用/不适用+理由)→假设→执行(推到利用/证伪)→关联。

【产物硬要求】（缺列即未完成，不许填"未做/N/A"留空）
- 指纹表：组件 | 版本 | CVE比对结果(编号+适用性+验证结论，或"无法联网+需人工复核")
- JS 资产表：来源文件 | 抽取方式 | 接口/参数/敏感信息(标是否全量)
- 五跳表：每组件一行，"执行"列为 利用结果 或 证伪证据
- FUZZ 矩阵：每参数一行(注入/边界/污染/type confusion/越权)，带证据
- 权限绕过表：手法(JWT各项/XFF/调试头/路径方法) | 测试 | 结果
- 攻击面表：业务功能 | 适用漏洞类(含业务逻辑) | 适用性理由 | 五跳结论
- 每个"安全"结论必带证据，无证据的"安全"无效
- 自我批判每个缺口，必须当场补完并附结果，只列不补 = 未完成

【对齐闸】开始前先复述：目标边界 + 你要穷举的维度 + 你对"完成"的理解
+ 确认五项强制都会执行。我审过再开测。
```

## 追问扳手（AI 停在半成品时补刀）

- "这只是识别，**钻到利用或证伪**了吗？证据呢？"
- "这个组件 **CVE 搜了吗、验了吗**？编号和结论？不能联网就声明。"
- "JS **全量**啃了吗？大包跳过没？路由表解析了？矩阵补全了？"
- "这个参数 **FUZZ 跑全**了吗（注入/边界/污染/type confusion/越权）？"
- "权限绕过四类（**JWT/XFF/调试头/路径方法**）都试了吗？"
- "CVE 编号**可溯源**吗？是不是只比了版本号、没看目标实测特征？"
- "**业务逻辑层**穷举了吗（越权/改价/状态机/竞态/SSRF）？还是只跑了技术指纹？"
- "你这是不是又在'铺得好看'？哪个点其实没钻到底？"
