# 人机协作渗透测试 SOP

> 长期方法论资产。范式：**AI 是知识源，你是导演 + 审计**。
> 仅用于授权范围内的安全评估。
> 版本 v1.2 · 2026-06-30

## 三层结构

| 层 | 文件 | 用途 | 变更频率 |
| --- | --- | --- | --- |
| L1 方法论 | `L1-methodology.md` | 给人读：原理、双轴、铁律。地基。 | 很少 |
| L2 指令模板 | `L2-ai-prompt.md` | 给 AI 吃：粘贴即跑的实战 prompt。 | 常微调 |
| L3 经验库 | `L3-knowledge.md` | 回灌：漏报案例、新打法、攻击链模式。 | 每项目 |

## 怎么用

1. 新项目 → 打开 `L2`，改目标名，粘给 AI 开测。
2. 测中 → 对照 `L1` 审 AI 输出（漏没漏、深不深）。
3. 收尾 → 把"AI 哪里漏了 / 什么新招有效"写进 `L3`。

## 迭代规则

- L3 只增不删，是复利发动机。
- L3 里反复出现的模式 → 提炼进 L1/L2。
- 改 L1 强制项 → 同步 L2 强制段（L1 为唯一事实源，L2 自包含以便粘贴即跑）。
- 重大改动打版本号 + 日期，便于回溯。

## 一句话

纵轴 8 阶段定"做什么"，横轴 4 方法定"怎么和 AI 干"。
人不当漏洞字典，当会逼问、会审计、会拍板的指挥者。
