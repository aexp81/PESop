# 人机协作渗透测试 SOP

> 长期方法论资产。范式：**AI 是知识源，你是导演 + 审计**。
> 仅用于授权范围内的安全评估。
> 版本 v3.3 · 2026-07-09。

## 项目地图（先看有哪些面）

这套 SOP 由三个内容文件构成三个面，外加本文件这个索引面。四个面不是并列
罗列，是**单向依赖链**：原则面产出规则 → 执行面消费规则 → 经验面在执行中
产生新发现 → 新发现反过来修订原则面，形成复利。

| 面 | 文件 | 承载什么 | 面向谁 | 变更频率 |
| --- | --- | --- | --- | --- |
| 索引面 | `README.md` | 怎么用、项目地图、版本历史 | 人 | 低 |
| 原则面 | `L1-methodology.md` | 四条根本准则、LLM 五种失效、验收清单——**唯一事实源** | 人（审计时对照） | 很少 |
| 执行面 | `L2-ai-prompt.md` | 粘给 AI 的可执行指令：动作清单 / 阶段闸 / 产物硬要求 | AI（每轮读取） | 常微调 |
| 经验面 | `L3-knowledge.md` | 每个项目收尾回灌的踩坑记录，复利资产 | 人（收尾写、下次开工读） | 每项目 |

## 业务流程（自上而下：从原则落到一次真实测试）

```
L1 原则  ──唯一事实源──▶  L2 执行  ──粘给 AI──▶  真实测试 0~7 阶段
  ▲                                                    │
  │                                                    ▼
  └──────────────  提炼 ◀── L3 经验 ◀── 回灌 ─────────┘
```

一次真实项目里，四个面按这个顺序咬合：

1. **开局**：打开 `L2`，改目标名，把「对齐闸」+「工作标准锚点」粘给 AI——
   工作标准锚点是 `L1` 四条准则的结论摘要，AI 不需要读 L1 全文。
2. **执行**：AI 按 `L2` 的阶段闸 0→7 推进，动作引用 `L2` 的 HF-1~7 编号；
   编号背后"为什么必须做"记在 `L1`，AI 不需要，人审计时需要。
3. **审计**：对照 `L1`《LLM 五种根本失效》F1-F5 检查 AI 输出，用 `L2`
   《追问扳手》逐条堵漏——L1 定义"什么算错"，L2 提供"怎么问出来"。
4. **收尾**：把"AI 这次漏了什么 / 什么新招有效"写进 `L3`（复制回灌模板）。
5. **迭代**：`L3` 里反复出现 ≥2 次的模式，先提炼进 `L1`（改事实源）再回灌
   `L2`（同步执行面），版本号 +1，记入下方版本历史。

## 迭代规则

- L3 只增不删，是复利发动机。
- L3 里反复出现的模式 → 提炼进 L1/L2。
- **L1 是唯一事实源**：新原则 / 新失效分类必须先落进 L1，再回灌 L2；
  禁止只改 L2 不回填 L1。
- 改 L1 强制项 → 同步 L2 动作清单段（L2 自包含以便粘贴即跑，但只放结论，
  不放推导；推导只在 L1 出现一次）。
- 重大改动打版本号 + 日期，记入下方版本历史，便于回溯。
