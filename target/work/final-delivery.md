# 最终交付（纯接口测试环境）

## 1. 测试完成状态

- 已完成：鉴权重放、IDOR/越权、审核绕过、自审自批、状态机跳步、SQLi/FUZZ、接口利用链。
- 已完成：基于真实业务 ID 的落库效果验证（非仅接口返回成功）。
- 未完成项：无（当前授权和环境条件下）。

## 2. 最高优先级漏洞（立即修）

1) Gw-S 可重放且跨接口复用（VULN-01）
- 风险：任意历史签名可长期复用，直接进入受保护写接口。

2) 审核/状态接口自审自批（AP-04/AP-05/AP-06）
- 风险：攻击者可伪造 `userId/userAd` 修改审核结果、取消原因、作废并删除记录。

3) 理赔流程节点跳过（AP-07）
- 风险：`Confirm` 可先于 `Review` 成功执行，状态机约束失效。

4) 队列重试控制可外部触发（AP-03）
- 风险：可触发重试/推送动作，造成流程扰动和消息放大。

## 3. 攻击剧本（最短路径）

1) 获取任意历史请求中的 `Nonce-Gw-S/Timestamp-Gw-S/Sign-Gw-S`。
2) 调用列表接口拿真实 ID（如 `CareAuditSnSearchPage`、`ActivationList`、`ActPushInsuranceInfo`）。
3) 用伪造 `userId/userAd` 调用审核/状态接口。
4) 复查列表接口确认状态/备注/记录确实被改写或删除。

## 4. 修复验收清单（上线前必须全绿）

- [ ] Gw-S 时间窗 <= 60s，nonce 单次消费，重放必拒绝。
- [ ] Gw-S 绑定 `method+path+query+body hash+caller identity`。
- [ ] 服务端忽略客户端提交的 `userId/userAd`，统一从认证上下文取操作者。
- [ ] 审核/状态机接口增加状态迁移白名单（from->to），违规迁移拒绝。
- [ ] `Confirm` 必须校验 `Review` 已完成且审批人/角色满足策略。
- [ ] 队列/任务接口下沉内网，仅调度系统可调用（mTLS + source allowlist）。
- [ ] 高风险写操作开启审计日志（操作者、源IP、请求摘要、前后值快照）。
- [ ] 对关键写接口设置速率限制和幂等键。

## 5. 复测通过标准

- 旧签名回放全部返回 `401/403`。
- 伪造 `userId/userAd` 不再影响记录，且审计日志记录真实操作者。
- `Review` 未完成时调用 `Confirm` 必须拒绝。
- 随机 ID/非归属 ID 的写请求必须返回对象不存在或无权，不得 `value:true`。
